# VSC v1.11 — GitHub Pages Publishing

## Purpose

VSC v1.11 publishes the static VSC showcase to GitHub Pages. The showcase is generated locally or in GitHub Actions and deployed as a public static site with no backend required.

---

## Public URL

```
https://digiemu.github.io/vsc-core/
```

---

## Publishing Model

- The showcase is generated by `npm run vsc -- showcase` (locally or in CI).
- Only `showcase/` is deployed to GitHub Pages.
- Heavy output folders (`output/recovery-*/`, `output/delta-*/`, `output/chain-*/`) are never deployed.
- `test-wp/` is never deployed.
- The site is static HTML, SVG, and CSS — no backend, no server required.
- The GitHub Actions workflow rebuilds the showcase on every push to `main`.

---

## GitHub Settings

To enable GitHub Pages for this repository:

1. Open [DigiEmu/vsc-core Settings → Pages](https://github.com/DigiEmu/vsc-core/settings/pages)
2. Under **Build and deployment**, set **Source** to **GitHub Actions**
3. Save
4. Push `.github/workflows/pages.yml` to `main`
5. Open the **Actions** tab and confirm the workflow runs successfully
6. The deployed URL will appear in Settings → Pages once the first deployment completes

---

## Commands

Generate the showcase locally before pushing:

```
npm run vsc -- demo:run
npm run vsc -- showcase
```

The GitHub Actions workflow runs both steps automatically on push to `main`.

---

## Workflow Summary

The `pages.yml` workflow:
1. Checks out the repository
2. Installs Node.js 22 and dependencies
3. Runs `npm run vsc -- demo:run` to generate token artifacts
4. Runs `npm run vsc -- showcase` to build `showcase/`
5. Uploads `showcase/` as a GitHub Pages artifact
6. Deploys to GitHub Pages

---

## Safety

The workflow and showcase export are designed to never publish:

- `output/recovery-*/` — recovery chunk folders
- `output/delta-*/` — restored delta working directories
- `output/chain-*/` — restored chain working directories
- `test-wp/` — local WordPress-style fixture
- Secrets or `.env` files
- `node_modules/`

---

## Status

Research prototype / MVP proof-of-concept, not production-ready.

The published site communicates the VSC proof-of-concept visually and technically. It does not represent a production service.
