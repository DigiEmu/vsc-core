# VSC v2.14 — PASS / FAIL Explained for Non-Technical Readers

Release: v2.14  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.14 adds a focused plain-language explanation of PASS, FAIL, and ERROR result classes.

The new document helps non-technical readers understand what each result means and what it does not mean.

## Added

New non-technical documentation:

```text
docs/non-technical/PASS_FAIL_EXPLAINED.md
```

## Key messages

```text
PASS means integrity checks passed.
PASS does not mean truth was proven.
```

```text
FAIL means an integrity problem was detected.
FAIL does not by itself prove wrongdoing.
```

```text
ERROR means verification could not be completed.
ERROR is not the same as FAIL.
```

## Purpose

This document helps readers avoid common misunderstandings:

- PASS is not a truth certificate.
- FAIL is not automatic proof of guilt or manipulation.
- ERROR is not the same as FAIL.
- VSC does not prove identity, legal responsibility, fairness, or attribution.
