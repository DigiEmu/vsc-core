# VSC v2.27 — Enterprise Review Package

Release: v2.27  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.27 adds an enterprise-facing review package for technical, governance, academic, partner, and due-diligence reviewers.

The package gives reviewers a single entry point for what to inspect, which commands to run, what outputs to expect, and how to interpret VSC proof boundaries.

## Added

New review document:

```text
docs/review/ENTERPRISE_REVIEW_PACKAGE.md
```

## Review package includes

- one-sentence VSC summary
- PASS / FAIL / ERROR interpretation
- validation commands and expected outputs
- key files to inspect
- technical verification path
- conformance / CI inspection points
- human-readable documentation map
- interop documentation map
- implemented / draft / examples-only / planning-only status categories
- reviewer checklist
- suggested reviewer questions
- explicit non-claims
- review outcome template

## Important status

This is a review package.

It is not certification.

v2.27 does not add:

- verifier logic
- CLI commands
- CI workflow changes
- fixture files
- fixture validation
- schema changes
- external reference validation
- external artifact validation
- integration-complete claims
- v3.0 start

## Expected validation commands

The review package references the existing validation commands:

```text
npm run vsc -- compare:fixtures
npm run vsc -- compare:fixtures --json
npm run vsc -- verify-all
```

Expected high-level outputs include:

```text
Final result: COMPARE_PASS
FAIL: 0
```

## Unchanged

v2.27 does not change:

- Go verifier behavior
- Node runner behavior
- CI workflow behavior
- conformance fixture expected results
- external reference schema semantics
- PASS / FAIL / ERROR technical meaning
- evidence bundle verification logic

## Release meaning

v2.27 packages the current VSC Core 2.x state for external review without expanding VSC's proof claims.
