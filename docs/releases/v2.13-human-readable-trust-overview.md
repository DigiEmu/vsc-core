# VSC v2.13 — Human-Readable Trust Overview

Release: v2.13  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.13 adds a plain-language trust overview for non-technical readers.

The new document explains what VSC is, what it proves, what it does not prove, and how PASS / FAIL / ERROR should be understood at a high level.

## Added

New non-technical documentation:

```text
docs/non-technical/VSC_TRUST_OVERVIEW.md
```

## Purpose

The goal of this release is to make VSC understandable for readers who are not developers.

This includes:

- partners
- auditors
- institutional readers
- governance stakeholders
- investors
- legal or policy-oriented readers

## Key message

```text
VSC proves state integrity, not truth.
```

## Boundary clarity

The overview clearly states that VSC does not prove:

- truth
- identity
- legal guilt
- responsibility attribution
- fairness
- moral correctness

VSC verifies the integrity and reproducibility of decision-state evidence bundles.

## Three-layer model

The document also explains the three-layer model:

```text
TBN     = identity / trust verification
VSC     = decision-state verification
CLARIXO = responsibility / attribution context
```

German short form:

```text
TBN beweist Identität.
VSC beweist Zustand.
CLARIXO beweist Verantwortungskontext.
```

## Unchanged

v2.13 does not change:

- Go verifier semantics
- Node runner behavior
- CI workflow behavior
- Conformance fixture expected results
- PASS / FAIL / ERROR technical meaning
- Evidence bundle verification logic

## Release meaning

v2.13 adds a human trust layer above the existing technical verification stack.

It helps VSC communicate clearly without expanding its technical or legal claims.
