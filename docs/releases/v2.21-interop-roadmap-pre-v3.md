# VSC v2.21 — Interop Roadmap / Pre-v3 Planning

Release: v2.21  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.21 adds a pre-v3 interoperability roadmap.

The roadmap consolidates the external reference and interop work completed in v2.17 through v2.20 and defines safe future phases before any possible v3.0 transition.

## Added

New roadmap document:

```text
docs/interop/INTEROP_ROADMAP_PRE_V3.md
```

## Completed interop arc

The roadmap summarizes:

```text
v2.17 — TBN / VSC / CLARIXO boundary notes
v2.18 — External Reference Profile Draft + JSON Schema
v2.19 — External Reference Examples
v2.20 — External Reference Validation Notes
```

## Key boundaries

```text
VSC proves state integrity, not truth.
```

```text
External references are pointers, not proof imports.
```

```text
Reference validation is not artifact validation.
```

## Future phases described

The roadmap describes possible future phases:

- optional reference-shape validation
- optional external reference conformance examples
- optional reporting integration
- optional partner handoff profiles
- optional external verifier handoff

These are planning phases only.

## v3.0 gate

The roadmap defines a v3.0 gate.

A move toward v3.0 should only be considered if boundaries, schema, examples, validation notes, reporting language, and partner handoff patterns are stable.

v3.0 has not started.

## Unchanged

v2.21 does not change:

- Go verifier behavior
- Node runner behavior
- CI workflow behavior
- conformance fixture expected results
- external reference schema semantics
- PASS / FAIL / ERROR technical meaning
- evidence bundle verification logic

## Implementation status

This release does not implement external reference validation.

It is documentation and planning only.

## Release meaning

v2.21 pauses implementation expansion and consolidates the interoperability path before any future verifier responsibilities are considered.
