# VSC v2.20 — External Reference Validation Notes

Release: v2.20  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.20 adds validation notes for the external reference profile introduced in v2.18 and illustrated in v2.19.

The notes clarify what future validation could safely check and what must remain outside VSC.

## Added

New interop documentation:

```text
docs/interop/EXTERNAL_REFERENCE_VALIDATION_NOTES.md
```

## Key message

```text
Reference validation is not artifact validation.
```

VSC may validate the shape of an external reference in the future.

That is not the same as validating the external artifact or its proof claim.

## Core boundary

```text
External references are pointers, not proof imports.
```

## Safe future validation scope

Future validation may check:

- required fields
- allowed enum values
- `proof_boundary = external_reference_only`
- `verified_by_vsc = false`
- JSON Schema shape
- pointer consistency at the reference-object level

## Out of scope

VSC does not validate:

- TBN receipt truth or validity
- CLARIXO record truth or validity
- external audit conclusions
- partner evidence trustworthiness
- issuer authority
- identity claims
- attribution claims
- compliance claims
- legal responsibility
- fairness
- truth

- README interop link

## Unchanged

v2.20 does not change:

- Go verifier behavior
- Node runner behavior
- CI workflow behavior
- conformance fixture expected results
- external reference schema semantics
- PASS / FAIL / ERROR technical meaning
- evidence bundle verification logic

## Implementation status

This release does not implement external reference validation.

It documents safe future validation boundaries only.

## Release meaning

v2.20 protects the VSC proof boundary before future validation work begins.

It clearly separates reference shape checks from external artifact proof claims.
