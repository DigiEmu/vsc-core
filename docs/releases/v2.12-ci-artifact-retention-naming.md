# VSC v2.12 — CI Artifact Retention / Naming Hardening

Release: v2.12  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.12 hardens CI artifact naming and retention for the conformance workflow.

The downloadable comparison artifacts introduced in v2.10 now include the GitHub Actions run number and commit SHA in the artifact name.

## Changed

CI artifact name changed from a static name:

```text
vsc-comparison-artifacts
```

to a run-aware and commit-aware name:

```text
vsc-comparison-artifacts-run-${{ github.run_number }}-${{ github.sha }}
```

## Added

Explicit artifact retention:

```yaml
retention-days: 30
```

## Artifact contents

The artifact still contains:

```text
comparison-result.json
comparison-report.md
```

## CI behavior

The workflow still validates:

```powershell
npm run vsc -- compare:fixtures
npm run vsc -- compare:fixtures --json
npm run vsc -- verify-all
```

CI still fails unless the JSON comparison result reports:

```json
{
  "result": "COMPARE_PASS"
}
```

## Unchanged

v2.12 does not change:

- Go verifier semantics
- Node runner comparison semantics
- GitHub Actions validation logic
- Conformance fixture expected results
- PASS / FAIL / ERROR meaning
- Evidence bundle verification logic

## Release meaning

v2.12 improves traceability of CI conformance artifacts by tying each uploaded artifact to a specific workflow run and commit.

This supports easier inspection, archive handling, and audit-style review of VSC conformance runs.
