# VSC v2.15 — Executive One-Pager / Partner Overview

Release: v2.15  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.15 adds a concise executive one-pager for partners, institutions, investors, and governance stakeholders.

The document explains VSC as a lightweight evidence layer for AI decision-state verification.

## Added

New partner-facing document:

```text
docs/non-technical/EXECUTIVE_ONE_PAGER.md
```

## Target readers

The one-pager is intended for:

- academic partners
- research groups
- funding readers
- investors
- AI governance partners
- audit and compliance readers
- non-technical decision-makers

## Key message

```text
VSC turns AI decision states into portable, verifiable evidence.
```

## Boundary statement

The document keeps the same strict boundary as the rest of VSC:

```text
VSC proves state integrity, not truth.
```

It does not claim that VSC proves truth, identity, legal guilt, responsibility attribution, fairness, or full compliance by itself.

## Three-layer model

The one-pager includes the broader trust architecture:

```text
TBN     = identity / trust verification
VSC     = decision-state verification
CLARIXO = responsibility / attribution context
```

## Unchanged

v2.15 does not change:

- Go verifier semantics
- Node runner behavior
- CI workflow behavior
- Conformance fixture expected results
- PASS / FAIL / ERROR technical meaning
- Evidence bundle verification logic

## Release meaning

v2.15 creates a concise partner-facing explanation layer above the technical and non-technical documentation.

It helps VSC communicate its value clearly while preserving strict proof boundaries.
