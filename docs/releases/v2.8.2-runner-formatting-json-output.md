# VSC v2.8.2 — Runner Table Formatting and JSON Output

Release: v2.8.2  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.8.2 improves the comparison runner introduced in v2.8.1.

This release makes the human-readable comparison table easier to inspect and adds an optional machine-readable JSON output mode for automation and future CI usage.

## Added

- Optional JSON output mode:

```powershell
npm run vsc -- compare:fixtures --json
```

## Improved

- Human-readable comparison table formatting
- Clearer fixture comparison output
- More automation-friendly runner behavior

## Unchanged

- Go verifier semantics
- Fixture expected results
- PASS / FAIL / ERROR semantics
- Comparison principle:

```text
same fixture input → same verification result class
```

## Validation

```powershell
npm run vsc -- compare:fixtures
npm run vsc -- compare:fixtures --json
npm run vsc -- verify-all
```

Expected:

```text
compare:fixtures        → Final result: COMPARE_PASS
compare:fixtures --json → "result": "COMPARE_PASS"
verify-all              → FAIL: 0
```

## Release meaning

v2.8.2 turns the comparison runner into a more readable and automation-ready tool.

It prepares the path for:

```text
v2.9 — CI Conformance Check
```
