# VSC v2.18 — External Reference Profile Draft

Release: v2.18  
Project: DigiEmu VSC / VSC Core

## Summary

VSC v2.18 adds a draft external reference profile.

The profile defines a safe, machine-readable way for VSC bundles, reports, or related metadata to reference external artifacts such as TBN receipts, CLARIXO records, audit notes, or partner evidence records.

## Added

New interop documentation:

```text
docs/interop/EXTERNAL_REFERENCE_PROFILE_DRAFT.md
```

New JSON Schema draft:

```text
schemas/external-reference-profile-v2.18-draft.schema.json
```

## Core boundary

```text
External references are pointers, not proof imports.
```

A VSC external reference points to an external artifact.

It does not mean VSC verified the external artifact.

It does not mean VSC absorbed or endorsed the external artifact's proof claims.

## Required draft fields

The draft profile defines:

- profile
- reference_id
- reference_type
- target_system
- target_artifact_id
- relation
- proof_boundary
- verified_by_vsc

## Strict draft constraints

The schema requires:

```text
proof_boundary = external_reference_only
verified_by_vsc = false
```

## Unchanged

v2.18 does not change:

- Go verifier behavior
- Node runner behavior
- CI workflow behavior
- conformance fixture expected results

- evidence bundle verification logic

## Integration status

v2.18 does not claim completed integration between TBN, VSC, and CLARIXO.

It defines a draft reference structure for safe future interoperability discussions.

## Release meaning

v2.18 turns the v2.17 boundary model into a first draft machine-readable reference profile.

It helps external artifacts be referenced without importing their proof claims into VSC.
